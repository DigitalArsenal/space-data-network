Getting Started

Install the SDS schemas and start building space data applications in minutes.

Installation

Install SDS schemas via your preferred package manager:

npm
npm install @spacedatastandards/schemas
Go
go get github.com/DigitalArsenal/spacedatastandards
Python
pip install spacedatastandards

Quick Example

Parse an Orbit Mean-Elements Message (OMM):

TypeScript
import { OMM } from '@spacedatastandards/schemas';
import Ajv from 'ajv';

const ajv = new Ajv();
const validate = ajv.compile(OMM.jsonSchema);

const data = {
  OBJECT_NAME: "ISS (ZARYA)",
  OBJECT_ID: "1998-067A",
  EPOCH: "2024-01-15T12:00:00Z",
  MEAN_MOTION: 15.49,
  ECCENTRICITY: 0.0001,
  INCLINATION: 51.6,
  RA_OF_ASC_NODE: 210.5,
  ARG_OF_PERICENTER: 45.2,
  MEAN_ANOMALY: 315.8
};

if (validate(data)) {
  console.log("Valid OMM message");
}

JSON Schema Format

JSON Schema is the primary format with x-flatbuffer annotations for binary serialization.

JSON Schema as Primary Format

All SDS schemas are defined as JSON Schema with custom x-flatbuffer-* annotations that map to FlatBuffer binary serialization. This gives you the best of both worlds: human-readable validation with JSON Schema, and high-performance binary serialization with FlatBuffers.

x-flatbuffer Annotations Reference

AnnotationDescriptionExample
x-flatbuffer-typeMaps JSON type to FlatBuffer type"double", "uint32", "string"
x-flatbuffer-field-idStable field ID for binary compatibility1, 2, 3
x-flatbuffer-deprecatedMarks field as deprecatedtrue
x-flatbuffer-defaultDefault value for field0, "UTC"
x-flatbuffer-requiredField is required in binary formattrue
x-flatbuffer-enumReferences a FlatBuffer enum type"timeSystem"

FlatBuffers Guide

Zero-copy binary serialization for high-performance space data exchange.

Why FlatBuffers?

FlatBuffers provides zero-copy deserialization, making it ideal for high-throughput space data streams. Data can be accessed directly from the binary buffer without parsing, reducing latency and memory allocation.

Key Benefits

  • Zero-copy access - Read data directly from binary buffer
  • Schema evolution - Add fields without breaking existing code
  • Cross-platform - Generated code for 14+ languages
  • Compact - Smaller than JSON, comparable to Protocol Buffers

Compiling Schemas

bash
# Generate TypeScript code from FlatBuffer schema
flatc --ts -o ./generated/ schemas/sds/schema/OMM/main.fbs

# Generate Go code
flatc --go -o ./generated/ schemas/sds/schema/OMM/main.fbs

# Generate Python code
flatc --python -o ./generated/ schemas/sds/schema/OMM/main.fbs

Integration Guide

Connect SDS schemas to the Space Data Network for real-time data exchange.

Space Data Network Integration

SDS schemas are the data format layer of the Space Data Network (SDN), a decentralized peer-to-peer network for space situational awareness.

SDN Data Flow

Architecture
Producer                    Network                     Consumer
   |                          |                           |
   |-- Serialize (FlatBuf) -->|                           |
   |                          |-- IPFS PubSub broadcast ->|
   |                          |                           |-- Validate (JSON Schema)
   |                          |                           |-- Deserialize (zero-copy)
   |                          |                           |-- Process data

API Reference

Schema registry, validation, and code generation APIs

Schema Registry API

GET /api/schemas

List all schemas with metadata

GET /api/schemas/{name}

Get schema metadata by name

GET /api/schemas/{name}/json-schema

Get JSON Schema with x-flatbuffer annotations

GET /api/schemas/{name}/flatbuffers

Get FlatBuffers .fbs file

GET /api/schemas/{name}/typescript

Get generated TypeScript types

GET /api/schemas/{name}/versions

List all versions of a schema

GET /api/schemas/{name}@{version}

Get a specific version of a schema

Example Response

GET /api/schemas/OMM
{
  "name": "OMM",
  "description": "Orbit Mean-Elements Message",
  "version": "1.0.5",
  "category": "orbital",
  "fieldCount": 42,
  "formats": ["json-schema", "flatbuffers", "typescript", "go", "python", "rust"],
  "links": {
    "json_schema": "/api/schemas/OMM/json-schema",
    "flatbuffers": "/api/schemas/OMM/flatbuffers",
    "typescript": "/api/schemas/OMM/typescript"
  }
}

Validation API

POST /api/validate

Validate data against a schema

Example Request

POST /api/validate
{
  "schema": "OMM",
  "data": {
    "OBJECT_NAME": "ISS (ZARYA)",
    "OBJECT_ID": "1998-067A",
    "EPOCH": "2024-01-15T12:00:00Z",
    "MEAN_MOTION": 15.49,
    "ECCENTRICITY": 0.0001,
    "INCLINATION": 51.6
  }
}

Example Response

200 OK
{
  "valid": true,
  "errors": [],
  "schema": "OMM",
  "version": "1.0.5"
}

Generation API

POST /api/generate

Generate code from a schema

Example Request

POST /api/generate
{
  "schema": "OMM",
  "format": "typescript"
}
